# AGENTS.md

This file provides guidance to AI Agents when working with code in this repository.

**All project documentation lives in `project_docs/`.** Do not create docs elsewhere.

```
project_docs/
├── architecture/    # ADRs and system architecture notes
├── database/        # Schema decisions, migration plans, SQL seeds
├── design/          # Feature specs, design decisions, UX notes
└── *.md             # Top-level deployment guides and one-pagers
```

When an agent creates a spec, plan, design doc, or any reference document, it must be written to the appropriate `project_docs/` subdirectory:

- Architecture decisions → `project_docs/architecture/`
- Feature specs and design docs → `project_docs/design/`
- Database-related docs → `project_docs/database/`

**Do not** create `docs/`, `docs/superpowers/`, or any other documentation directory at the repo root.

---

## Agent rules (for multi-model workflows)

When asked to plan or implement any design, UI or UX tasks please use the 'ui-ux-pro-max' skill.

When a coding agent completes a feature, it should leave clear notes in the relevant `project_docs/design/` spec about what was built, any deviations from the spec, and what the testing agent should verify.

Do not put "Generated by AGENT" in a PR. This is a team effort.
